# Codex Token Usage

[![Codex plugin](https://img.shields.io/badge/Codex-plugin-10A37F)](https://developers.openai.com/codex/plugins/build)
[![License: AGPL v3](https://img.shields.io/badge/license-AGPL_v3-blue.svg)](LICENSE)
[![Dependencies](https://img.shields.io/badge/dependencies-bash%20%2B%20jq-4EAA25)](#requirements)

See the tokens used by each Codex prompt-to-response turn. No API calls, telemetry, or estimation: the plugin reads the usage snapshots already stored in Codex's local session JSONL.

## Showcase

```text
Codex's response ends here.
warning: 15:57:37 | turn total: 16,129 tokens (fresh input 11,574, cache read 4,480, output 75, reasoning 67)
```

Codex currently renders hook `systemMessage` output with a `warning:` prefix. That prefix is a UI convention, not a hook failure.

## Install

### From GitHub

```bash
codex plugin marketplace add wilmtang/codex-token-usage
codex plugin add codex-token-usage@codex-token-usage
```

Start a new task and approve the hook when Codex asks you to trust it.

### From a local clone

```bash
git clone https://github.com/wilmtang/codex-token-usage.git
codex plugin marketplace add ./codex-token-usage
codex plugin add codex-token-usage@codex-token-usage
```

## Requirements

- Codex with plugin and hook support
- Bash
- [`jq`](https://jqlang.org/download/)

On macOS: `brew install jq`. On Debian or Ubuntu: `sudo apt install jq`.

## How it works

Codex passes the `Stop` hook a session path or session ID. The hook:

1. Opens only the matching local session JSONL.
2. Finds the cumulative usage snapshot immediately before your latest prompt.
3. Subtracts it from the latest snapshot after the response.
4. Prints one comma-formatted summary through the hook's `systemMessage` field.

| Field | Meaning |
| --- | --- |
| `fresh input` | Input tokens not served from prompt cache |
| `cache read` | Input tokens reused from prompt cache |
| `output` | Tokens generated by the model; reasoning is included here |
| `reasoning` | The reasoning-token subset of output |
| `turn total` | Input plus output across all model calls in the turn |

`turn total` is processed token volume, not context-window size, plan usage, or a dollar-cost estimate. It can be large when one turn makes many model calls or repeatedly reads a large cached context.

Everything runs locally. The hook reads only the matching Codex session file and makes no network requests.
